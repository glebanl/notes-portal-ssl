# Notes Portal SSL

## 项目简介

`notes-portal-ssl` 是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面通常作为笔记、信息展示或轻量级工具的入口，不针对任何特定域名或网站。本仓库旨在提供一个整洁、可维护的静态页面集合，方便快速查阅和共享。

## 目录说明

```
notes-portal-ssl/
├── index.html          # 主入口页面（如导航或索引）
├── pages/              # 存放各独立 HTML 页面的子目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 资源文件（CSS、JS、图片等）
│   ├── styles/
│   ├── scripts/
│   └── images/
└── README.md           # 本文件
```

- `index.html`：仓库的默认首页，通常用于提供页面列表或链接。
- `pages/`：所有独立 HTML 页面存放于此，每个文件代表一个独立内容单元。
- `assets/`：页面共享或专用的静态资源，按类型组织。

## 页面归档说明

- 每个页面均为独立完整的 HTML 文件，可直接在浏览器中打开运行。
- 页面内容以笔记、说明或工具形式为主，不包含动态后端依赖。
- 归档遵循简洁、清晰的原则，避免冗余或重复内容。
- 所有页面均通过 HTTPS 协议提供访问，确保传输安全（若部署于支持 SSL 的平台）。

## 维护说明

- 添加新页面：在 `pages/` 目录下创建新 HTML 文件，并建议在 `index.html` 中添加链接。
- 更新内容：直接编辑对应 HTML 文件，保持文件结构一致。
- 删除页面：移除对应文件并更新索引（如有）。
- 资源管理：公共资源放于 `assets/` 根目录，页面专用资源可放于子文件夹。
- 提交说明：建议以简洁描述记录每次变更，如“添加页面：XXX”、“更新页面：XXX 内容”。

## 使用方式

1. 克隆仓库到本地。
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任一文件。
3. 若部署到静态托管服务，确保根目录指向仓库根目录。
